**📌 프로젝트 개요**

- Eclipse와 Oracle DB를 연결하여 자바 콘솔에서 메일을 송수신하는 서비스

- 프로젝트 기간: 2024년 4월 10일 ~ 2024년 4월 24일
- 프로젝트 인원: 개인
- 사용 언어: JAVA, MySQL
- 개발 환경: Eclipse, Oracle, DBeaver

- MVC 패턴 구현을 위해 UI, DAO, VO, SERVICE 클래스로 구성

**📌 구현 기능**

- 회원가입 및 로그인, ID/PW 찾기, 회원정보 수정

- 회원가입 시 ID 중복 체크, 비밀번호 일치 여부 체크
- 로그인 5회 실패 시 본인인증을 통해 비밀번호 재설정
- 비밀번호 확인 후 비밀번호/휴대폰번호 변경 가능

- 받은메일함 메일 조회, 전달, 답장

- 받은메일함에서 단어를 검색하면 발신자, 제목, 내용에 해당 단어가 포함된 메일 조회
- ‘전달’ 메뉴를 통해 타인에게 메일 제목과 내용 그대로 전송 가능
- ‘답장’ 메뉴를 통해 수신인 자동 세팅

- 보낸메일함 수신확인, 전송취소

- 보낸 메일을 조회하고 수신자의 메일 열람 여부 확인 가능
- 수신자가 메일을 읽기 전이라면 보낸 메일 전송 취소 가능

- 휴지통 비우기, 복원

- 메일함에서 ‘휴지통으로이동’ 메뉴를 통해 메일을 휴지통으로 이동
- ‘삭제’ 메뉴를 통해 휴지통을 거치지 않고 바로 삭제도 가능
- 휴지통에서 ‘복원’ 메뉴를 통해 받은메일함으로 복구 가능

**📌 ER 다이어그램**

![메일_DB](https://github.com/user-attachments/assets/fa38b800-4887-4840-8959-a8daa494fbf5)

- 회원정보(T_MEMBER) 테이블

- 회원번호(NO)를 PK로 가지는 멤버 테이블을 생성하고, ID에 Unique 조건 추가
- 비밀번호(PW), 이름(NAME), 생년월일(BIRTH), 휴대폰번호(HP_NUM), 가입일(REG_DATE) 컬럼 생성
- 가입일에 DEFAULT값으로 SYSDATE 속성 추가

- 메일함(T_MAIL) 테이블

- 일련번호(NO)를 PK로 가지는 메일 테이블을 생성하고,
   일련번호를 FK로 가지는 받은메일함, 보낸메일함, 휴지통 테이블 생성
- READYN 컬럼을 추가하고 DEFAULT를 0으로 세팅
   → DAO에서 메일 조회 시 해당 값을 1로 UPDATE

### **🐶 동물병원 진료 예약 서비스(Myungranghan Dog)**

![DOG_MAIN](https://github.com/user-attachments/assets/22b92dc1-87f9-40ba-ba5f-fdbdaff0db8d)

**📌 프로젝트 개요**

- 강아지 동물병원 소개 및 예약, 진료정보 등을 조회하는 동물병원 웹 사이트

- 프로젝트 기간: 2024년 6월 17일 ~ 2024년 6월 30일
- 프로젝트 인원: 개인 및 팀(3인)
- 사용 언어: JAVA, HTML, CSS, JavaScript, MyBatis
- 개발 환경: Eclipse, Tomcat, Oracle, DBeaver

- 각자의 웹페이지를 만들고 3명의 팀원이 DB Link를 통해 주요 테이블을 공유하여
한 아이디로 반려견 종합 서비스를 제공
- SpringBoot를 사용하지 않고 MyBatis를 이용하여 SQL Query문을 처리

**📌 구현 기능**

- 회원가입 및 로그인

- DB Link를 이용하여 회원정보, 견종, 반려견정보를 담고 있는 테이블을 공유
   →하나의 아이디로 각자가 개발한 웹 사이트의 서비스를 이용할 수 있도록 구현
- 로그인 시 마이페이지/로그아웃 메뉴 노출
- 로그아웃 시 회원가입/로그인 메뉴 노출

- 사용자: 진료 예약 및 예약 정보 조회/변경/취소

- 통합 페이지에서 회원가입 후 로그인 시 등록한 반려견 정보 자동 세팅
- 원하는 날짜와 시간 선택 후 예약 완료 시 마이페이지에서 예약 정보 조회/변경/취소 가능
- 예약이 중복된 날짜를 선택했을 경우 재선택 요구 메시지창 노출

- 관리자: 진료정보 등록 및 조회/삭제

- 회원정보 테이블의 admin 컬럼의 값이 ‘A’인 사용자는 관리자로 인식하고,
   마이페이지에서 강아지 진료기록 정보 등록 가능
- 회원 아이디 입력 후 조회 버튼 클릭 시 해당 회원의 강아지 진료기록 조회/삭제 가능

**📌 ER 다이어그램**

![DOG_SDB](https://github.com/user-attachments/assets/f7933f96-7a6e-4f72-9329-9ff7367addfd)

![DOG_DB](https://github.com/user-attachments/assets/5b89dd4b-727f-485f-9890-f0fb66e23794)

- 회원정보(MEMBER), 반려견정보(DOG), 견종정보(BREED) 테이블을 DB Link를 통해 공유

- 각 테이블마다 중복되는 이름(NAME), 생년월일(BIRTH) 등의 컬럼명을 구별하기 위해 각 테이블명을 컬럼명 앞에 추가
- 회원정보 테이블의 PK를 반려견정보의 FK로 설정
- 견종정보 테이블의 견종코드(BREED_CODE)를 반려견정보의 FK로 설정
- 회원정보의 ADMIN 컬럼의 DEFAULT 값을 ‘U’로 세팅하고, 운영자일 경우 ‘A’ 로 변경되도록 구성
